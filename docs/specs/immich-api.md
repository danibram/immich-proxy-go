# Immich API Reference

This document describes the Immich API endpoints used by the proxy for shared link functionality.

## Overview

The proxy communicates with Immich using its REST API. For shared links, Immich uses either:
- **Share Key**: A random string generated by Immich (e.g., `AbCdEf123456...`)
- **Slug**: A custom human-readable alias (e.g., `my-wedding`)

## Authentication for Shared Links

Shared links don't require user authentication. Instead, they use query parameters:

| Parameter | Description |
|-----------|-------------|
| `key` | The share key (Immich-generated) |
| `slug` | Custom URL alias (if configured) |
| `password` | Password for protected shares |

Some endpoints also accept headers for compatibility:
- `x-immich-share-key`: Share key or slug
- `x-immich-share-password`: Password

## Endpoints Used by Proxy

### Get Shared Link Info

```
GET /api/shared-links/me?key={key}
GET /api/shared-links/me?slug={slug}
GET /api/shared-links/me?key={key}&password={password}
```

Returns information about the shared link including permissions and assets.

**Response:**
```json
{
  "id": "uuid",
  "key": "AbCdEf123...",
  "type": "ALBUM",
  "expiresAt": "2024-12-31T23:59:59.000Z",
  "allowUpload": false,
  "allowDownload": true,
  "showMetadata": true,
  "album": {
    "id": "album-uuid",
    "albumName": "My Album",
    "assetCount": 150,
    "assets": [...]
  },
  "assets": [...]
}
```

**Status Codes:**
- `200` - Success
- `401` - Invalid password or share key
- `404` - Share not found

---

### Get Album Details

```
GET /api/albums/{albumId}?key={key}
GET /api/albums/{albumId}?slug={slug}
```

Returns full album details including all assets.

**Response:**
```json
{
  "id": "album-uuid",
  "albumName": "My Album",
  "description": "Album description",
  "assetCount": 150,
  "assets": [
    {
      "id": "asset-uuid",
      "type": "IMAGE",
      "originalFileName": "photo.jpg",
      "fileCreatedAt": "2024-01-15T10:30:00.000Z",
      "isTrashed": false,
      "exifInfo": {...}
    }
  ]
}
```

---

### Get Thumbnail

```
GET /api/assets/{assetId}/thumbnail?key={key}&size={size}
GET /api/assets/{assetId}/thumbnail?slug={slug}&size={size}
```

**Parameters:**
- `size`: `preview` (larger) or `thumbnail` (smaller, default)

**Response:** Binary image data (JPEG/WebP)

**Headers:**
- `Content-Type`: `image/jpeg` or `image/webp`

---

### Get Original File

```
GET /api/assets/{assetId}/original?key={key}
GET /api/assets/{assetId}/original?slug={slug}
```

Downloads the original full-resolution file.

**Response:** Binary file data

**Headers:**
- `Content-Type`: Original MIME type (e.g., `image/jpeg`, `video/mp4`)
- `Content-Disposition`: `attachment; filename="original_filename.ext"`

---

### Get Video Playback

```
GET /api/assets/{assetId}/video/playback?key={key}
GET /api/assets/{assetId}/video/playback?slug={slug}
```

Streams video for playback. Supports range requests.

**Headers:**
- `Range`: Optional byte range for seeking

**Response:** Video stream

---

### Upload Asset

```
POST /api/assets?key={key}
POST /api/assets?slug={slug}
```

Upload a new asset to the shared album. Requires `allowUpload: true` on the share.

**Headers:**
- `Content-Type`: `multipart/form-data`
- `x-immich-share-key`: Share key

**Form Fields:**
- `assetData`: File binary
- `deviceAssetId`: Unique identifier (can be filename + timestamp)
- `deviceId`: Device identifier
- `fileCreatedAt`: ISO timestamp
- `fileModifiedAt`: ISO timestamp

**Response:**
```json
{
  "id": "new-asset-uuid",
  "duplicate": false
}
```

---

### Add Asset to Album

```
PUT /api/albums/{albumId}/assets?key={key}
PUT /api/albums/{albumId}/assets?slug={slug}
```

Associates an uploaded asset with the album.

**Headers:**
- `Content-Type`: `application/json`
- `x-immich-share-key`: Share key

**Body:**
```json
{
  "ids": ["asset-uuid"]
}
```

## Data Types

### SharedLink

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier |
| `key` | string | Share key |
| `type` | string | `ALBUM` or `INDIVIDUAL` |
| `expiresAt` | string? | ISO timestamp, null if no expiration |
| `allowUpload` | boolean | Whether uploads are permitted |
| `allowDownload` | boolean | Whether downloads are permitted |
| `showMetadata` | boolean | Whether EXIF data is visible |
| `password` | string? | Set if password-protected |
| `album` | Album? | Album object if type is ALBUM |
| `assets` | Asset[] | Direct assets (for INDIVIDUAL type) |

### Asset

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier |
| `type` | string | `IMAGE`, `VIDEO`, `AUDIO`, `OTHER` |
| `originalFileName` | string | Original filename |
| `originalMimeType` | string | MIME type |
| `fileCreatedAt` | string | ISO timestamp |
| `isTrashed` | boolean | Whether asset is in trash |
| `isFavorite` | boolean | Whether asset is favorited |
| `duration` | string | Video duration (e.g., "0:01:30.000") |
| `exifInfo` | ExifInfo? | EXIF metadata |
| `thumbhash` | string | BlurHash for placeholder |

### ExifInfo

| Field | Type | Description |
|-------|------|-------------|
| `make` | string | Camera manufacturer |
| `model` | string | Camera model |
| `exifImageWidth` | number | Image width in pixels |
| `exifImageHeight` | number | Image height in pixels |
| `fileSizeInByte` | number | File size |
| `fNumber` | number | Aperture (f-stop) |
| `focalLength` | number | Focal length in mm |
| `iso` | number | ISO sensitivity |
| `exposureTime` | string | Shutter speed |
| `latitude` | number | GPS latitude |
| `longitude` | number | GPS longitude |
| `city` | string | Location city |
| `state` | string | Location state |
| `country` | string | Location country |

## Error Responses

Immich returns errors in this format:

```json
{
  "message": "Human-readable error message",
  "error": "Error type",
  "statusCode": 401
}
```

Common errors:
- `401 Invalid share key` - Share key doesn't exist
- `401 Invalid password` - Wrong password
- `404 Not found` - Asset or album not found
- `403 Forbidden` - Action not permitted (e.g., download disabled)

## Notes

1. **Immich validates share access**: All requests with a share key are validated by Immich. It only returns assets that belong to that share.

2. **Large albums**: The `/api/shared-links/me` endpoint may not return all assets for very large albums. Use `/api/albums/{id}` to get the complete asset list.

3. **Trashed assets**: Immich may include trashed assets in responses. The proxy filters these out.

4. **Expiration**: Immich checks `expiresAt` server-side. The proxy also validates expiration as a double-check.

5. **Password hashing**: Passwords are compared server-side by Immich. The proxy passes them through.
