# API Specification

## Base URL

All share-related endpoints support two URL formats:

| Format | Description | Example |
|--------|-------------|---------|
| `/share/{key}/api/...` | Standard Immich share keys | `/share/AbCdEf123.../api/shared-links/me` |
| `/s/{slug}/api/...` | Custom URL aliases (slugs) | `/s/my-wedding/api/shared-links/me` |

**Key vs Slug:**
- **Key** (`/share/`): The random identifier generated by Immich (e.g., `JUckRMxlgpo7F9...`)
- **Slug** (`/s/`): Custom alias configured in Immich when creating a shared link (e.g., `my-wedding`)

The proxy automatically detects which format is being used and communicates with Immich accordingly.

## Authentication

Password-protected shares require authentication via:
- Cookie: `immich-share-password` (signed value)
- Header: `X-Immich-Share-Password` (raw password)

## Endpoints Overview

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Health check |
| GET | `/shared-links/me` | Get share info and assets |
| POST | `/shared-links/me/password` | Validate password |
| GET | `/albums/{albumId}` | Get album details |
| GET | `/assets/{assetId}` | Get asset details (EXIF, filename) |
| GET | `/assets/{assetId}/thumbnail` | Get thumbnail image |
| GET | `/assets/{assetId}/thumbnail.{ext}` | Thumbnail with extension (CDN-cacheable form) |
| GET | `/assets/{assetId}/original` | Download original file |
| GET | `/assets/{assetId}/video/playback` | Stream video |
| POST | `/assets/download` | Start bulk download job |
| GET | `/download/jobs/{jobId}` | Get download job status |
| GET | `/download/jobs/{jobId}/file` | Download completed ZIP |
| POST | `/assets` | Upload file |
| POST | `/upload-check` | Pre-upload checksum dedupe check |
| GET | `{share}/raw` | Raw bytes of an INDIVIDUAL image share (embeddable) |
| GET | `{share}/og-cover` | Album cover for link unfurls (no Sec-Fetch guard) |

## Endpoints

### Health Check

```
GET /health
GET /healthcheck
```

**Response**: `200 OK`
```json
{"status": "ok"}
```

---

### Get Shared Link Info

```
GET /share/{key}/api/shared-links/me
```

**Response**: `200 OK`
```json
{
  "id": "uuid",
  "key": "share-key",
  "type": "ALBUM",
  "expiresAt": "2024-12-31T23:59:59Z",
  "allowUpload": false,
  "allowDownload": true,
  "showMetadata": true,
  "album": {
    "id": "uuid",
    "albumName": "My Album",
    "assets": [...]
  },
  "assets": [...]
}
```

**Errors**:
- `401 Unauthorized`: `{"passwordRequired": true}`
- `404 Not Found`: Share not found
- `410 Gone`: Share expired

---

### Validate Password

```
POST /share/{key}/api/shared-links/me/password
Content-Type: application/json
```

**Request**:
```json
{"password": "secret"}
```

**Response**: `200 OK`
```json
{"valid": true}
```

Sets `immich-share-password` cookie on success.

**Errors**:
- `401 Unauthorized`: Invalid password
- `429 Too Many Requests`: Rate limited

---

### Get Album

```
GET /share/{key}/api/albums/{albumId}
```

**Response**: `200 OK`
```json
{
  "id": "uuid",
  "albumName": "My Album",
  "assets": [...],
  "assetCount": 42
}
```

---

### Get Thumbnail

```
GET /share/{key}/api/assets/{assetId}/thumbnail?size=thumbnail|preview|fullsize
```

```
GET /share/{key}/api/assets/{assetId}/thumbnail.webp?size=thumbnail
GET /share/{key}/api/assets/{assetId}/thumbnail.jpg?size=preview
```

**Response**: Image binary with appropriate `Content-Type`

The extensioned form is what the web client uses: the extension makes the URL
eligible for CDN default caching (Cloudflare caches by extension). Only `webp`
and `jpg` are accepted; the extension is advisory — `Content-Type` wins.

`fullsize` is available only when `options.max_zoom_quality: fullsize` and the
Immich shared link allows downloads. It is never publicly cached.

### Raw Individual Image

```
GET /share/{key}/raw
GET /s/{slug}/raw
```

Returns image bytes only for a single-image `INDIVIDUAL` share. Password and
expiry rules still apply. This URL can be embedded directly in an `<img>`.

---

### Get Original File

```
GET /share/{key}/api/assets/{assetId}/original
```

**Response**: File binary with `Content-Disposition: attachment`. Image quality
is capped by `options.max_download_quality`; videos remain original.

**Access Control**: Downloads blocked if:
- Proxy config `options.allow_download: false`
- Shared link has `allowDownload: false` in Immich

**Errors**:
- `403 Forbidden`: Downloads disabled (proxy or shared link level)

---

### Get Video

```
GET /share/{key}/api/assets/{assetId}/video/playback
```

**Response**: Video stream. The incoming `Range` header is forwarded to
Immich and `206 Partial Content` / `Content-Range` / `Accept-Ranges` pass
through, so browsers can seek. Streaming has no total deadline (an idle
watchdog aborts transfers that stop making progress).

---

### Bulk Download (ZIP) - Job-based

Bulk downloads use a job system with progress tracking.

#### Start Download Job

```
POST /share/{key}/api/assets/download
Content-Type: application/json
```

**Request**:
```json
{
  "assetIds": ["uuid-1", "uuid-2", "uuid-3"]
}
```

**Response**: `200 OK`
```json
{
  "jobId": "abc123def456..."
}
```

**Constraints**:
- Minimum 2 assets (for single file use `/original`)

**Errors**:
- `400 Bad Request`: Invalid input
- `403 Forbidden`: Downloads disabled (proxy or shared link level)

---

#### Get Download Job Status

```
GET /share/{key}/api/download/jobs/{jobId}
```

**Response**: `200 OK`
```json
{
  "id": "abc123def456...",
  "status": "processing",
  "progress": 45,
  "total": 100,
  "filename": "My Album.zip"
}
```

**Status values**:
- `processing`: ZIP is being generated
- `ready`: ZIP is ready for download
- `failed`: Generation failed (see `error` field)

**Errors**:
- `404 Not Found`: Job not found or expired

---

#### Download Completed ZIP

```
GET /share/{key}/api/download/jobs/{jobId}/file
```

**Response**: ZIP file with `Content-Disposition: attachment`

**Errors**:
- `404 Not Found`: Job not found
- `409 Conflict`: Job not ready yet

---

### Upload Asset

```
POST /share/{key}/api/assets
Content-Type: multipart/form-data | image/* | video/*
x-immich-checksum: <sha1 hex|base64>   (optional)
```

When `x-immich-checksum` is sent and the asset already exists in the link
owner's library, Immich short-circuits with `200 {"id": "...", "status":
"duplicate"}` before the body is consumed.

**Response**: `201 Created`
```json
{
  "id": "uuid",
  "status": "created"
}
```

**Errors**:
- `403 Forbidden`: Upload not allowed for this share
- `413 Request Entity Too Large`: File too large
- `415 Unsupported Media Type`: Invalid file type

### Pre-upload Check

```
POST /share/{key}/api/upload-check
Content-Type: application/json

{"files": [{"name": "IMG_0001.jpg", "checksum": "<sha1 hex|base64>"}]}
```

**Response**: `200 OK`
```json
{"results": [{"name": "IMG_0001.jpg", "checksum": "...", "exists": true, "assetId": "uuid"}]}
```

Lets the client skip sending bytes for files already in the album (the web
client hashes in a Web Worker and calls this once per batch; up to 500 items).
Fails open: on upstream trouble every file reports `exists: false`.
Implementation notes: `docs/specs/immich-api.md`.

---

## Error Responses

All errors return plain text body with appropriate HTTP status code:

| Status | Meaning |
|--------|---------|
| 400 | Bad Request - Invalid input |
| 401 | Unauthorized - Password required |
| 403 | Forbidden - Action not allowed |
| 404 | Not Found - Resource doesn't exist |
| 410 | Gone - Share expired |
| 413 | Payload Too Large - Upload too big |
| 415 | Unsupported Media Type - Invalid file type |
| 429 | Too Many Requests - Rate limited |
| 500 | Internal Server Error |
